# gql-fiber

Lightweight GraphQL server in Go using gqlgen and Fiber (example project).

## Overview

- Implements a GraphQL API generated by `gqlgen`.
- Schema files: `internal/graphql/schema/`
- Resolvers: `internal/graphql/resolver/`
- Generated code: `internal/graphql/generated/`

## Prerequisites

- Go 1.18+ (recommended latest stable)
- `gqlgen` (you can run it via `go run github.com/99designs/gqlgen@v0.17.86 generate`)

## Setup

Clone and fetch modules:

```bash
git clone <repo-url> gql-fiber
cd gql-fiber
go mod download
```

Generate code (when you change schemas):

```bash
go run github.com/99designs/gqlgen@v0.17.86 generate
```

## Run

Start the server:

```bash
go run ./cmd/server
```

The server listens on `:8080`.

- GraphQL endpoint: http://localhost:8080/query
- GraphQL Playground: http://localhost:8080/

## Example GraphQL operations

Login (mutation):

```graphql
mutation Login($input: LoginInput!) {
  login(input: $input) {
    accessToken
  }
}

# Example variables
{
  "input": {
    "email": "test@example.com",
    "password": "123456"
  }
}
```

User (query):

```graphql
query GetUser($id: ID!) {
  user(id: $id) {
    id
    name
    email
  }
}
```

Users (query):

```graphql
query {
  users {
    id
    name
    email
  }
}
```

Curl example (POST to GraphQL endpoint):

```bash
curl -X POST \
  -H "Content-Type: application/json" \
  --data '{"query":"query{users{id name email}}"}' \
  http://localhost:8080/query
```

## Project structure

- `cmd/server` — server entrypoint
- `internal/graphql/schema` — GraphQL schema files
- `internal/graphql/resolver` — resolver implementations
- `internal/graphql/model` — GraphQL model types
- `internal/graphql/generated` — code generated by `gqlgen`

## Development notes

- Edit schema files under `internal/graphql/schema` and re-run `gqlgen generate`.
- Resolver stubs are generated; implement business logic in `internal/graphql/resolver`.
